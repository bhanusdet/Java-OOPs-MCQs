🔹 Object-Oriented Programming Concepts

Core OOPs principles and their relationships

Encapsulation vs Abstraction

FeatureEncapsulationAbstraction
Primary FocusHides data (variables and internal state)Hides implementation details (how methods work)
What it hidesData members, internal variablesMethod implementations, complex logic
How it's achievedAccess modifiers (private, protected), getters/settersAbstract classes, interfaces, method signatures
PurposeData security, controlled access, data integrityShow only essential features, reduce complexity
Level of controlControls access to dataControls exposure of functionality
Implementation bindingTightly bound to data structureLoosely bound - focuses on contracts
FlexibilityLess flexible - tied to specific dataMore flexible - multiple implementations possible
Design focusData-centric - protecting informationBehavior-centric - defining contracts
Real-life analogyMedicine capsule hides the drug insideCar dashboard hides engine complexity
Code exampleprivate String name; public getName()abstract void draw(); interface Shape{}

Inheritance vs Polymorphism

FeatureInheritancePolymorphism
Primary FocusCode reuse by deriving new classes from existing onesMultiple forms - same interface, different behaviors
Relationship TypeStructural relationship (is-a relationship)Behavioral relationship (behaves-as relationship)
How it's achievedextends (class) / implements (interface)Method Overloading (compile-time) & Method Overriding (runtime)
PurposePromote reusability, establish hierarchical relationshipsFlexibility, dynamic behavior at runtime
Code ImpactReduces redundancy, improves maintainabilityIncreases extensibility, promotes loose coupling
Binding TimeCompile-time (static relationship)Compile-time (overloading) + Runtime (overriding)
DependencyChild depends on parent structureCaller depends on interface, not implementation
Design PatternTemplate Method, Strategy (with inheritance)Strategy, Factory, Command patterns
Real-life analogyChild inherits traits from parent (eye color, height)Person acts as student, employee, customer (same person, different roles)
Code exampleclass Dog extends AnimalAnimal a = new Dog(); a.sound();
