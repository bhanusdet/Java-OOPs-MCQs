üîç Find Command Mastery - Your File Detective Tool

üí° Think of 'find' as your personal detective that searches the entire file system based on clues you provide.

üéØ Find by Name Pattern

find /home -name "*.log"

What this does: Searches in /home directory and all subdirectories for files ending with .log

Real example output:
/home/john/app.log
/home/john/projects/server.log
/home/mary/debug.log

find . -iname "*README*"

Case-insensitive search: Finds README, readme, ReadMe, etc. The -i flag ignores case

üóÇÔ∏è Find by File Type

find /var -type f

File types explained:
‚Ä¢ f = regular files (documents, images, programs)
‚Ä¢ d = directories (folders)
‚Ä¢ l = symbolic links (shortcuts)
‚Ä¢ b = block devices (hard drives)
‚Ä¢ c = character devices (keyboard, mouse)

find /etc -type d -name "*conf*"

Practical example: Find all directories in /etc that contain "conf" in their name

üìè Find by Size - Clean Up Space Hogs

find /home -size +100M

Size operators:
‚Ä¢ +100M = larger than 100MB
‚Ä¢ -1M = smaller than 1MB
‚Ä¢ 50M = exactly 50MB
‚Ä¢ +1G = larger than 1GB

find /var/log -size +50M -ls

Space cleanup scenario: Find large log files and show detailed info with -ls

‚è∞ Find by Time - Track Changes

find /var/log -mtime -7

Time types:
‚Ä¢ -mtime = modification time (when content changed)
‚Ä¢ -atime = access time (when file was read)
‚Ä¢ -ctime = change time (when permissions/ownership changed)

Time values:
‚Ä¢ -7 = within last 7 days
‚Ä¢ +30 = older than 30 days
‚Ä¢ 7 = exactly 7 days ago

find /home -mtime -1 -type f

Real scenario: "What files were modified in the last 24 hours?"

üîß Execute Commands on Results - Batch Processing

find /tmp -name "*.tmp" -delete

‚ö†Ô∏è DANGEROUS but useful: Finds and immediately deletes all .tmp files. BE CAREFUL!

find /home -name "*.jpg" -exec cp {} /backup/ \;

How -exec works:
‚Ä¢ {} = placeholder for each found file
‚Ä¢ \; = end of command (backslash escapes the semicolon)
‚Ä¢ This copies every .jpg file to /backup/

find /var/log -name "*.log" -exec gzip {} \;

Log management: Compress all log files to save space

find . -name "*.c" -exec grep -l "main" {} \;

Code search: Find all C files that contain the word "main"

üéØ Combining Conditions - Power User Mode

find /home -name "*.log" -size +10M -mtime +30

Complex search: Find log files larger than 10MB that are older than 30 days

find /var -type f \( -name "*.log" -o -name "*.tmp" \) -delete

Logical OR: Find files that are EITHER .log OR .tmp and delete them

üìù Text Processing Tools - Data Manipulation Masters

üîç GREP - The Text Hunter

üí° Think of grep as Ctrl+F for the entire file system

grep -r "error" /var/log/

Recursive search explanation:
‚Ä¢ -r = search in all files in all subdirectories
‚Ä¢ Searches every file under /var/log/ for the word "error"
‚Ä¢ Shows filename and line containing the match

grep -n "pattern" file.txt

Line numbers: Shows which line number contains the match

Case-insensitive: Finds "error", "ERROR", "Error", etc.

Exclude matches: Show lines that DON'T contain "debug", first 20 lines

Count matches: Just shows how many lines contain "failed"

üé® AWK - The Column Processor

üí° AWK treats every line as columns separated by spaces/tabs

awk '{print $1, $3}' file.txt

Column selection:
‚Ä¢ $1 = first column, $2 = second column, etc.
‚Ä¢ $0 = entire line
‚Ä¢ NF = number of fields (columns)
‚Ä¢ $NF = last column

ps aux | awk '{print $1, $2, $11}' | head -10

Real example: Show username, PID, and command from process list

awk '{sum+=$2} END {print "Total:", sum}' numbers.txt

Calculate totals:
‚Ä¢ Adds up all values in column 2
‚Ä¢ END section runs after processing all lines
‚Ä¢ Perfect for calculating sums, averages

awk '$3 > 100 {print $1, $3}' data.txt

Conditional processing: Only print lines where column 3 is greater than 100

awk -F',' '{print $1, $3}' data.csv

CSV processing: -F',' sets comma as field separator for CSV files

‚úèÔ∏è SED - The Text Editor

üí° SED is like find-and-replace for the command line

sed 's/old/new/g' file.txt

Replacement anatomy:
‚Ä¢ s = substitute command
‚Ä¢ /old/new/ = replace "old" with "new"
‚Ä¢ g = global (all occurrences on each line)
‚Ä¢ Without 'g', only first occurrence per line is replaced

sed -i 's/localhost/server.com/g' config.txt

Edit in-place: -i modifies the file directly instead of showing output

sed -n '10,20p' file.txt

Extract lines: Print only lines 10 through 20

sed '/^#/d' config.conf

Remove comments: Delete all lines starting with # (configuration files)

sed 's/^[ \t]*//' file.txt

Remove leading spaces: Clean up indentation

üîó Combining Tools - Pipeline Power

grep "error" /var/log/*.log | awk '{print $1, $4}' | sort | uniq -c

Complex pipeline:
1. Find all lines with "error" in log files
2. Extract timestamp and error type
3. Sort the results
4. Count unique occurrences

ps aux | awk '$3 > 5.0 {print $1, $2, $3, $11}' | sort -k3 -nr

System monitoring: Find processes using more than 5% CPU, sort by CPU usage

sed -i 's/old/new/g' *.txt

Replace in-place in all .txt files

üîê Permissions & Security - Understanding Who Can Do What
üé≠ Understanding Linux Permissions - The Three-Act Play

üí° Think of permissions like a theater:

  • Owner = Director (full control)
  • Group = Cast members (shared access)
  • Others = Audience (limited access)

üìä Reading Permission Strings

-rwxr-xr-- 1 john staff 1024 Jan 10 10:30 script.sh

Breaking down: -rwxr-xr--

Position Character Meaning
1 - File type (- = file, d = directory, l = link)
2-4 rwx Owner permissions (read, write, execute)
5-7 r-x Group permissions (read, no write, execute)
8-10 r-- Others permissions (read only)

üî¢ Number System (The Permission Calculator)

Each permission has a number value:

  • Read (r) = 4
  • Write (w) = 2
  • Execute (x) = 1

Common permission combinations:

7 = 4+2+1 = rwx (read, write, execute) - Full access 6 = 4+2 = rw- (read, write) - Can modify but not run 5 = 4+1 = r-x (read, execute) - Can read and run but not modify 4 = 4 = r-- (read only) - Can only view 0 = 0 = --- (no access) - Cannot access at all

üéØ Real-world Permission Examples

chmod 755 script.sh

755 breakdown:
‚Ä¢ Owner: 7 (rwx) - can read, modify, and run the script
‚Ä¢ Group: 5 (r-x) - can read and run, but not modify
‚Ä¢ Others: 5 (r-x) - can read and run, but not modify
Perfect for: Executable scripts that everyone can run

chmod 644 document.txt

644 breakdown:
‚Ä¢ Owner: 6 (rw-) - can read and edit
‚Ä¢ Group: 4 (r--) - can only read
‚Ä¢ Others: 4 (r--) - can only read
Perfect for: Documents that you edit but others just read

chmod 600 private.key

600 breakdown:
‚Ä¢ Owner: 6 (rw-) - can read and edit
‚Ä¢ Group: 0 (---) - no access
‚Ä¢ Others: 0 (---) - no access
Perfect for: Private files like SSH keys, passwords

chmod 777 /tmp/shared/

‚ö†Ô∏è 777 - DANGEROUS but sometimes needed:
‚Ä¢ Everyone has full access (read, write, execute)
‚Ä¢ Use only for temporary directories
‚Ä¢ Never use on important files!

üë• User and Group Management - Building Your Team

üè† Understanding Ownership

chown john:developers project.txt

Ownership format: user:group
‚Ä¢ john = the owner (user)
‚Ä¢ developers = the group
‚Ä¢ Now john owns it, and anyone in developers group has group permissions

chown john project.txt

Change owner only: Keep existing group unchanged

chown :admin project.txt

Change group only: Keep existing owner, change group to admin

chown -R john:users /home/john/projects/

Recursive ownership: Change owner and group for directory and ALL contents

üé´ Sudo - Temporary Admin Powers

sudo apt install software

How sudo works:
1. Asks for YOUR password (not root's)
2. Checks if you're allowed to run admin commands
3. Runs the command as root
4. Goes back to normal user after command finishes

sudo -u john cp file.txt /home/john/

Run as different user: Execute command as 'john' instead of root

sudo -i

Become root permanently: Switch to root shell (be careful!)

sudo !!!

Repeat last command with sudo: If you forgot sudo, this reruns last command as root

üîÑ User Switching

su - john

Switch user completely:
‚Ä¢ - loads john's environment (home directory, PATH, etc.)
‚Ä¢ Asks for john's password
‚Ä¢ Type 'exit' to return to original user

su john

Switch user but keep environment: Become john but stay in current directory

üõ°Ô∏è Advanced Permission Features

chmod u+s /usr/bin/passwd

SUID (Set User ID):
‚Ä¢ When anyone runs this file, it runs with owner's permissions
‚Ä¢ Example: passwd command runs as root so users can change passwords
‚Ä¢ Shows as 's' in owner execute position: rwsr-xr-x

chmod g+s /shared/projects/

SGID on directory:
‚Ä¢ Files created in this directory inherit the directory's group
‚Ä¢ Great for shared project folders
‚Ä¢ Shows as 's' in group execute position: rwxrwsr-x

chmod +t /tmp/

Sticky bit on directory:
‚Ä¢ Users can only delete their own files
‚Ä¢ Even if they have write permission to directory
‚Ä¢ Shows as 't' in others execute position: rwxrwxrwt
‚Ä¢ Perfect for /tmp directory

üîç Permission Troubleshooting

ls -la problematic_file

Check current permissions: See who can do what

Check user's groups: See what groups user belongs to

id

Check your identity: See your user ID, group ID, and all groups

umask

Check default permissions: See what permissions new files get

umask 022

Set default permissions:
‚Ä¢ 022 means new files get 644 (rw-r--r--)
‚Ä¢ 022 means new directories get 755 (rwxr-xr-x)
‚Ä¢ Lower umask = more permissive defaults

Understanding Permissions

Permission format: rwxrwxrwx

  • First rwx = owner permissions
  • Second rwx = group permissions
  • Third rwx = other permissions
  • r=read(4), w=write(2), x=execute(1)

Common permission combinations:

chmod 755 script.sh

rwxr-xr-x (owner: read+write+execute, group/others: read+execute)

chmod 644 document.txt

rw-r--r-- (owner: read+write, group/others: read only)

chmod 600 private.key

rw------- (owner: read+write, group/others: no access)

Advanced Permission Management

Recursive permissions:

chmod -R 755 /var/www/

Apply permissions to directory and all contents

Special permissions:

chmod u+s /usr/bin/passwd

Set SUID bit (runs with owner's privileges)

chmod g+s /shared/directory

Set SGID bit (new files inherit group)

chmod +t /tmp

Set sticky bit (only owner can delete files)

Default permissions:

umask 022

Set default permissions (files: 644, directories: 755)

üëÅÔ∏è Viewing Processes - Who's Running What?

ps aux

Understanding the output columns:

USER PID %CPU %MEM VSZ RSS TTY STAT START TIME COMMAND
john 1234 2.5 3.2 45676 8192 pts/0 R+ 10:30 0:05 python3 script.py
root 5678 0.1 1.5 12345 4096 ? S 09:15 0:15 nginx: worker

Column explanations:

  • USER: Who owns this process
  • PID: Process ID number (unique identifier)
  • %CPU: Percentage of CPU being used right now
  • %MEM: Percentage of RAM being used
  • VSZ: Virtual memory size (total memory allocated)
  • RSS: Resident set size (actual RAM being used)
  • TTY: Terminal where process was started (? = background)
  • STAT: Process state (R=running, S=sleeping, Z=zombie)
  • START: When the process started
  • TIME: Total CPU time used
  • COMMAND: The actual command/program running
ps aux --sort=-%cpu | head -10

Find CPU hogs: Show top 10 processes using most CPU

ps aux --sort=-%mem | head -10

Find memory hogs: Show top 10 processes using most RAM

pgrep -f "python"

Find process by name: Get PID of all processes containing "python"

‚è∏Ô∏è Process Control Signals

Signals are like different ways to tap someone on the shoulder:

kill -15 1234

SIGTERM (15) - Polite request:
‚Ä¢ "Please close yourself properly"
‚Ä¢ Allows process to clean up, save data
‚Ä¢ Most processes will obey this
‚Ä¢ Default kill signal if none specified

kill -9 1234

SIGKILL (9) - Immediate termination:
‚Ä¢ "Die right now, no questions asked"
‚Ä¢ Cannot be ignored or blocked
‚Ä¢ Process cannot clean up
‚Ä¢ Use only when SIGTERM doesn't work

kill -1 1234

SIGHUP (1) - Reload configuration:
‚Ä¢ "Reload your config files"
‚Ä¢ Many server processes restart when receiving this
‚Ä¢ Web servers often reload config without stopping

kill -STOP 1234

SIGSTOP - Pause process:
‚Ä¢ Freezes process in place
‚Ä¢ Use kill -CONT 1234 to resume
